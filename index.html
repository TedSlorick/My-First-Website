Beginner HTML

Boilerplate

Before getting started with any HTML task we need to establish the boilerplate. The boilerplate establishes things to the web browser such as the DOCTYPE, we also need the <html> tags which are referred to as the root element of the document.

The root element is also where we establish the language using lang="en"

Within the root element is where we will find the head and body elements. The head contains such information as the metadata and page title. The body is where all of the content for our page is going to live.

Managing Text

Once the boilerplate has been established we can start to build out the content that we want displaying on our page.

If we were to just fill the body with text, no matter how we format the text within our text editor it will come out all bunched together. This is because we need to give the browser tags so it knows how to display the information.

To begin with we are just using the heading (h1-h6) tags to define our heirarchy and then the paragraph tag (<p>) to establish our paragraphs of text.

Within these we can provide some formatting to our content using the strong and emphasis tags. The strong tag makes the content within it bold whereas the emphasis tag (<em>) puts it in italics.

Very often you will see the use of <b> and <i> to create bold and italic text. The issue with this is that it is just a visual change and does not add any meaning to the text. The meaing provided by <strong> and <em> is vitally important for website accessibility if users are using tools such as a screen reader.

Lists

Lists are obviously incredibly useful, we have 2 main types ordered and unordered.

To create a basic unordered list we use the tag <ul> and each item in the list is created with the <li> tag. Every item within an unordered list will display as an item with a bullet point. See the example below:

  • This
  • is
  • an
  • unordered
  • list

An ordered list uses the <ol> tag and will create a numbered list. Items within the list are still created using the <li> tag. See example below:

  1. This
  2. is
  3. an
  4. ordered
  5. list

Links

Anchor Tags

In order to create a link we use the anchor element which has the tag <a>. However, just using the anchor tage will not do anything we need to use a HTML Attribute to tell it a destination.

An attribute is usually made up of two parts a name and a value. The attribute we use in this case is href (hypertext reference). Anchor tags don't just got to links they can also be used for videos, pdf files, images and most importantly other HTML documents.

Other Attributes

Some other attributes we might use with an anchor tag could be:

  • target = determines where to open link. _self value means same tab, _blank means new tab
  • rel = Describes the relation between the current page and linked resource. Two most relevant values:
    • noopener = prevents new tab from accessing original page
    • noreferrer = same as noopener but also prevents certain details about original page being shared.

Absolute and relative

Absolute links to pages on other websites and will always contain the scheme and domain of the desitination.

Relative links to other pages within your own site. They do not contain the domain name as they obviously dont need them. As our index.html is always going to be in the root directory it is best to make a new directory to store the pages we are going to be linking to. Here is a link to the lists page completed earlier: Lists

The anchor arguement looks like this <a href="./pages/lists.html">Lists</a>

The . makes the code start looking for the file/directory relative to the current directory. In this case pages is the name of the directory I made to store the pages and lists.html is the name of the document.

Images

We can embed images into our website using the <img> element. This is an example of a void element. We use the src="" attribute to define the location of the image. Like with links this can be absolute or relative. Again to improve accessibility we may choose to use the alt="" attribute which provides alternative text, used by screen readers. Usually we will end up using CSS to modify our images, however, it is still good practice to specify the height and width attributes using the actual size of the image.
